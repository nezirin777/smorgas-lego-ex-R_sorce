# smorgas lego-ex-R 変更履歴
----

## 2018/05/22
* Firefox ESR60で動作するように、HTMLのscriptタグでJavaScriptのバージョン指定を廃止しました。
	+ これに伴い、動作要件をFirefox 52以上に引き上げました。Pale Moon 26は動作不可となり、Pale Moon 27で使用するにはスクリプトのリビルドが必要です。
	+ コピー機能や新着の音を鳴らす際に Flashを使う方式は廃止しました。

* bbs2chreader ex対応を廃止しました。
	+ bb2ch: プロトコルを使わず、chaika:// に統一しました。
	+ b2rreload、b2raboneadd イベント対応の処理を削除しました。

* 5ch.net に対応しました。
	+ 5ch/2ch.net のどちらのURLでも板名を引けるように対応し、板名テーブルも更新しました。
	+ style-outlink.css で 5ch.netのスレッドURLのアイコン表示に対応し、アイコンも変更しました。
	+ スレッド情報ダイアログの「リンク」タブを「内部リンク」「外部リンク」に変更し、「内部リンク」にスレッドと同じドメイン内のリンクを抽出表示するようにしました。

* スキン設定などのデータを保持するためのブラウザ側の設定に関して、オプションダイアログの「補足説明」「FAQ」タブの説明を現状に即した内容にしました。 
	+ ブラウザの「ブラウザープライバシー設定」において、chaikaのローカルサーバーが動作するサイト（ポート番号含む）からの Cookie の保存を許可する必要があります。
	+ 履歴を消去する際に「Cookie」「ウェブサイトのオフライン作業用データ」にチェックを付けているとスキンのデータも消えます。
	+ スキンがデータを保存できないとスキン設定、しおり機能、自分/返信マーク機能、ReplaceStr.txt機能は動作しません。

* 「Cookie とサイトデータ」が許可されていなかった場合について考慮しました。
	+ 許可しない設定でも、動作中にサイトデータを使う機能は動作しないものの、スキン設定は options.js スクリプトの初期値を書き換えて読み込ますことは可能です。
	+ しかし、毎回エラー確認画面で複数回止められることになっていました。そこで、options.js にて SysPref.disableStorage を true に設定しておくと、エラーを出さないように処理をスキップし、関連機能を無効化するようにしました。

* スレッド情報ポップアップの表示が undefined になるバグを修正しました。

## 2017/07/16

* コピー機能でレス本文をコピーすると、レスアンカーやURLリンクの後ろに「false」が付いてしまうバグを修正しました。
  2017/01/10版でのケアレスミスでした。

## 2017/05/08

## 2017/05/07

  これにより、Pale Moon 27（26は不可）でも動作するようになりました。

## 2017/03/01
* 画像のインライン（埋め込み）表示を、使用メモリ削減のためにサムネイル画像に縮小してから
  表示するオプションを追加しました。
	+ 画像をサムネイルにする（初期値:有効）
	+ GIFは除外する（初期値:無効）

  画像をいったん`Image`オブジェクトに読み込み、縮小画像を`canvas`に描画して表示、その後元の
  Imageオブジェクトを破棄しています。メモリ開放はGC任せですので一気に複数画像の読み込み
  をすると__一時的にかえって使用メモリが増える可能性があります__のでご注意ください。

  そういうこともあり、遅延読込の同時読み込み数のパラメータを小さな値に調整しました。

	+ `valueLazyloadMax: 40` → `10`
	+ `valueLazyloadMin: 30` → `2`

  アニメーションGIFをサムネイル化した場合、__最初のフレームの静止画__になります。
  アニメーションはリンクからのポップアップでご覧ください。

* レス番号部のコンテキストメニューの「埋込」を「埋込（強制）」として、そのレスの
  埋め込み要素をいったんクリアしてから埋め込み処理を行うようにしました。
  埋め込み画像がうまく読み込めなかった場合のリトライにご利用ください。

* 2ちゃんねるやBBSPINKで「グロ」のキーワードによるグロ判定が効かなくなっていたので
  グロチェック条件（正規表現）のデフォルト値を以下のように修正しました。

  `(^ ?グロ|^ ?グロテスク|注意|危険|ブラクラ)([ 　!！\(（\:.．。だで]|$)`

  今回はグロチェック条件を強制的に初期化はしません。
  各自でオプションダイアログから上記のように修正するか、設定初期化でオプション全体を初期化する
  かしてください。

* ポップアップ表示されたレス中の画像・動画リンクの埋め込みやポップアップが正常に動作しなく
  なっていたバグを修正しました。

## 2016/11/28
* 埋め込み画像・動画の自動読み込みで「表示範囲外も読み込む」オプションを追加し、パラメータ
  の調整も行いました。

* 「自動で読み込む」が有効の場合に対象の埋め込み要素をいったん遅延読込予約リストに追加して
  から一定間隔で読み込み要求をかける方式は変えていません。（サーバー、ローカルの負荷を考慮）
  ただし、その間隔を200msecから50msecに変更しました。

* 予約リストへの追加は画面表示範囲から近い順にソートして行います。同時読み込み数上限値（50）
  まで追加したら、読み込み完了して読み込み数が下限値（40）を下回ったら予約リストに追加、
  というように表示範囲から順次読み込みを行っていきます。

* 「表示範囲外も読み込む」が無効の場合は表示範囲内に入った要素のみ読み込みを行います。

* `checkLazyload()`の動作を確認できるようにコンソールログを出すようにしておきました。

  スキンスレで上がっていた画像の多いスレの例ですが、2,000個以上も画像のあるスレを全レス
  表示して画像を全部読み込むというのは現実的ではないと思います。
  そもそも全部表示しようとしても保持できずにサムネイル画像が壊れるかと。
  そういう意味でも__見る分だけ表示させる利用法を推奨__します。

* オプションダイアログの「返信レスを強調表示する」に誤記があったのを修正しました。

* オプションの「ID を発言回数で色分けする」の初期値を有効に変更しました。

* `style-example.txt` の追加スタイル定義は現状に適合しないものや選択したスタイル依存であったりと
  混乱の元となるおそれがあるため削除しました。

* JSDoc 3 でドキュメントを作成する手順 `_JSDoc3` を更新しました。実行には `Node.js` が必要です。

## 2016/05/30

	+ レス左側のヘッダ情報部分の項目の並びはオリジナルに準拠してIDが先で名前が後にしています。

	+ ID発言数の状態表示アイコン用にHTMLの要素を追加し、colleライク使用時にはID色分けや逆参照情報を
	  属性セットする処理を行います。
	+ CSSで項目を柔軟にレイアウトするためレスヘッダ部の1階層余分なHTML構造を廃止しました。

* 検索ボックスで強調表示の場合もヒットした先頭のレスにスクロールする（オプション）ようにしました。

* 埋め込み画像・動画のスクロールによる自動読み込みが一部のスタイルで動作しないバグを修正しました。

  このためオプションは再度、初期状態から設定・保存が必要です。ただし、すでに以前のスキンで保存した
  オプションを上書きしないようにしていますので、スキンを切り替えて設定値を見比べることは出来ます。

* 3文字仕様のIDも発言回数や参照を処理し、`???` を含むIDは無視するようにしました。

* スクリプトにJSDoc 3用のドッグコメントを付けました。`_JSDoc3`フォルダをご覧ください。

* Firefoxの「前のページへ戻ります」ボタンでスレッド表示に戻った場合に不要なリロードが起きないように
  するため、オプションの「非アクティブ時でも自動更新を強制する」の初期値を`true`に変更しました。
  リロードしてもスクロール位置を復元したい場合はしおり機能を使ってください。

* あぼーんしたレスのNGDATAを要素のtitleにセットしてツールチップ表示するようにし、あぼーんレス表示
  切り替えは`ABONE` をクリックで行うようにしました。また、以前のあぼーん表示はキツすぎたので
  マイルドに調整しました。
  この変更のためにレスhtmlの構造を変更しました。既存の lego-ex用スタイルも変更が必要です。

* chaikaの即時あぼーんイベントに対応しました。NGExの場合はリロードします。

* スレッドタイトル部分のコンテキストメニューに「あぼーん切り替え」を追加し、URLに
  `?chaika_disable_abone=1`
  を付けたり取ったりしてリロードするようにしました。
