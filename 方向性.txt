# Chaika スキンスクリプト改良方針

## プロジェクト目的

機能追加は行わない。

主目的は以下。

- 可読性向上
- 保守性向上
- 軽量化
- パフォーマンス改善
- 現代Firefox向け最適化
- script.js巨大化問題の解消
- 将来的な機能追加や不具合修正を容易にする

動作仕様は原則変更しない。
既存利用者が違和感を覚えるようなUI変更や挙動変更は避ける。


# 対象環境

対象

- Firefox 最新版
- Floorp
- Zen Browser
- Waterfox

対象外

- Firefox ESR60以前
- Firefox 52
- Pale Moon
- 古いES5環境
- トランスパイル前提環境

そのため以下は削除候補。

- 古いブラウザ回避コード
- Polyfill
- ES5向け記述
- Pale Moon対応コード
- トランスパイル前提コード


# 基本方針

## 1. 動作を壊さない

リファクタリング優先。

× 機能追加
× UI変更
× 仕様変更

○ 同じ動作で整理
○ 同じ動作で軽量化
○ 同じ動作で高速化


## 2. namespace方式からClass方式へ移行

現状

var Popup = { ... };
var Bookmark = { ... };
var AutoReload = { ... };

目標

class Popup { ... }
class Bookmark { ... }
class AutoReload { ... }

一括変換は禁止。

必ず

解析
↓
最小修正
↓
動作確認
↓
Class化
↓
動作確認

の順で進める。


## 3. script.js分割を最終目標とする

現状

script.js
（巨大単一ファイル）

最終目標

core/
thread/
popup/
ui/
util/

など責務別に分離。

ただし初期段階では分割しない。

まず内部整理を優先する。


# Class化ルール

## 状態を持つモジュール

現状

var Bookmark = {
    _data: {},
    save(){},
    load(){}
};

↓

class Bookmark {

    constructor(){
        this._data = {};
    }

    save(){}

    load(){}
}

const bookmark = new Bookmark();


## 状態を持たないモジュール

現状

var UrlUtil = {
    normalize(){},
    encode(){}
};

↓

class UrlUtil {

    static normalize(){}

    static encode(){}

}


## thisの利用を明確化

曖昧な外部参照は避ける。

例

Popup._timer

↓

this.timer
