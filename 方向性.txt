# Chaika スキンスクリプト リファクタリング方針

## プロジェクト目的

機能追加は行わない。

目的は以下。

- 可読性向上
- 保守性向上
- 軽量化
- パフォーマンス改善
- 現代Firefox向け最適化
- script.js巨大化問題の解消
- 将来的な修正容易化

既存利用者から見た挙動は可能な限り維持する。


# 対象環境

対象

- Firefox 最新版
- Floorp
- Zen Browser
- Waterfox

対象外

- Firefox 52
- Firefox ESR旧版
- Pale Moon
- ES5環境

これら旧環境向け互換コードは削除候補。


# 現在の認識

このコードは完全な旧式コードではない。

実際には

- functionベース
- namespaceベース
- classベース

が混在している。

例

function AjaxGet()

var Nodes = {}

class StyleSheet {}

つまり

「全部Class化する」

ではなく

「古い部分を段階的に整理しながらClass化する」

方針を採用する。


# 基本原則

## 1. まず整理、次にClass化

禁止

- いきなり全面書き換え
- 一括Class化
- 一括ES化

推奨

解析
↓
最小修正
↓
動作確認
↓
Class化
↓
動作確認


## 2. 小さい単位で進める

毎回

1モジュール
または
1責務

単位で作業する。

大規模改修は禁止。


## 3. 動作変更を避ける

優先順位

保守性
＞
軽量化
＞
見た目変更

挙動変更は最後の手段。


# Class化方針

## 状態を持つもの

現状

var Bookmark = {
    _data: {},
    save(){},
    load(){}
};

↓

class Bookmark {

    constructor() {
        this._data = {};
    }

    save(){}

    load(){}
}

const bookmark = new Bookmark();


## 状態を持たないもの

現状

var UrlUtil = {
    normalize(){},
    encode(){}
};

↓

class UrlUtil {

    static normalize(){}

    static encode(){}

}


# DOM方針

現在のコードには

- _doc
- Nodes
- ResNodes

など独自管理層が存在する。

これらは全体への影響が大きいため後回し。

まずは利用側を整理する。

Nodes/ResNodesの改修は後半で実施する。


# 優先順位

## Phase1 現状把握

対象コードの責務確認

- 何を担当しているか
- 誰が呼んでいるか
- 何に依存しているか

を整理する。


## Phase2 安全な整理

対象

- const/let化
- 未使用変数削除
- 未使用関数削除
- 重複処理削除
- 古い互換コード削除
- 不要なDOM取得削除

目的

- サイズ削減
- 可読性向上


## Phase3 小規模Class化

優先

1. AjaxGet
2. StyleSheet周辺
3. 小規模ユーティリティ

ここでClass化のルールを固める。


## Phase4 Popup系整理

対象

- Popup
- ResPopup
- IDPopup
- NamePopup
- TrackbackPopup
- UrlPopup
- ImagePopup
- VideoPopup

目標

共通基底クラス

class PopupBase

を導入する。


## Phase5 UI系整理

対象

- Bookmark
- AutoReload
- Digest
- AutoScroll
- FindBox
- PageScroller

など

Class化と責務整理を行う。


## Phase6 Thread系整理

対象

- ThreadDocument
- Name
- ID
- Trackback
- ResNumber
- ResImage

依存が多いため後回し。


## Phase7 Nodes/ResNodes整理

対象

- Nodes
- ResNodes

理由

全体への影響が非常に大きい。

後半で慎重に実施する。

必要であれば

var Nodes

↓

class NodesManager

への移行も検討。


## Phase8 ファイル分割

最終目標

core/
popup/
thread/
ui/
util/

へ責務ごとに分割。

例

core/
 ├ AjaxGet.js
 ├ StyleSheet.js

popup/
 ├ PopupBase.js
 ├ UrlPopup.js
 ├ ImagePopup.js

ui/
 ├ Bookmark.js
 ├ AutoReload.js

thread/
 ├ ThreadDocument.js

util/
 ├ Clipboard.js
 ├ ReplaceStr.js


# レビュー時の重点確認項目

毎回確認するもの

- 古いFirefox向けコード
- Pale Moon向けコード
- querySelector重複
- querySelectorAll乱用
- setTimeout乱用
- setInterval乱用
- 不要なDOM再検索
- グローバル依存
- namespace肥大化
- 責務混在
- 未使用コード
- 重複コード


# 作業手順

毎回必ず

1. 現状コード確認
2. 責務整理
3. 依存関係確認
4. 問題点抽出
5. 最小修正版提示
6. 差分説明
7. 動作確認
8. Class化案提示
9. 差分説明
10. 動作確認

の順で進める。
